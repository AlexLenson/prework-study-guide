HTML

  • The head element contains information about the webpage.
  • Head element contains metadata that is not visible to the user
  • Metadata tells the browser how to render the page correctly
  • Title tag is visible to user (good title name is important for SEO!)
  • The body element represents the visible content shown to the user.
  • Body element is an example of a semantic HTML element. Semantic elements are element names that describe their content, use them often!
  • Common non-semantic elements include "div" (to create division on page) and "span" (inline division)
  • The "alt" attribute for the "img" element describes the image and is important for screen readers/SEO
  • The body element in "index.html" for this page is structured using "header", "main" and "footer" elements. The "main" element is divided into four sections using "section" elements.
  • Right now we are in the "section" element with attribute id="html-section"

CSS

  • CSS defines style of webpage (colors, aesthetics, visual layout, and organization)
  • Class and element selectors. We can style HTML elements using elements or class attributes as selectors
  • Internal vs external style sheets
  • Inline CSS: applies to specific HTML tag using a style attribute
  • Internal CSS is defined in HTML head section and applies to all tagged elements on that page only
  • External CSS is in a separate file and linked using "link" element in HTML head
  • CSS rule is broken down into selectors and declarations
  • Selector defines the element or attributes to which the rules, or declarations, will apply
  • CSS box model: every HTML element is a box with margin, border, padding and content
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • "Link" element creates connection to external file, "rel" attribute specifies relationship bw current and linked files, "href" specifies URL
  • Assign rules to speciifc HTML elements using class attributes and class selectors. Give HTML element a class attribute and use CSS class selector to style it!
  • Class selectors start with a period (ex: .card is a "card" class selector)

Git

  • Git is used to track versions of a project
  • GitHub allows you to store your project in a remote repository and collaborate with other developers
  • GitHub Workflow:
    • Make new feature branch (locally)
      • This is where we can save our "work in progress". Good practice to name branches "feature/(whatever the feature is)"
    • Add & commit changes (locally)
    • Before pushing, it's good practice to pull source code from github to ensure we have latest commits and changes & no conflicts bw GitHub and local repo:
      • "git pull origin main" (should say "Already up to date")
    • Push changes to feature branch on github (remote):
      • "git push origin feature/starter-code"
      • Note: Always push to your feature branch, not main branch. Essential for keeping bugs from breaking project
    • Can now see the repo we pushed changes to in github
    • Open PR (github)
      • Make sure correct base and compare branches are selected ("base: main" and "comare: feature/starter-code")
    • Merge into main branch
    • Main branch is now updated with your changes
    • Finally, update local main branch with new changes.
      • git checkout main
      • git pull origin main
    • Notes:
      • Don't code directly in main branch. Instead, make a copy (feature branch) and merge back into main branch.
      • Changes made to main branch are tracked by feature branches. Main branch (should be) the final, most updated version of the application.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • Git Flow Commands:
  • Stage changes: git add -A
  • Commit: git commit -m “your message here”
  • Pull github base branch into local feature branch to make sure local branch is up to date. Do this before pushing changes to github feature branch: git pull origin main
  • Push changes to remote feature branch. Note, make sure to push to the feature branch, not the main branch!: git push origin feature/name-of-branch
  • We can now open a PR, merge the branch and close the issue

JavaScript

  • JavaScript allows users to interact with websites (effects, interactivity, data communication, and much more)
  • JavaScript is an object oriented programming language (OOP)
  • JavaScript turns static HTML/CSS page into an interactive page!
  • Use "var" keyword to declare a variable
  • Use console.log() to print to web console. The word console refers to a test environment that developers use to check out their code.
  • To link JS file to HTML, add script element just before closing "body" tag: <script src="./assets/script.js"></script>"
  • An array holds a group of data (usually related in some way). Can store multiple data types in a single array
  • For loop contains three statements: starting point for our loop, condition, and increment the iterator
  • Use "function" keyword and function name to declare a function
  • JS has built-in Math object. Use "Math.floor() and Math.random()" to generate a random number
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.

Hotkeys

  • VSCode Hotkeys:
  • Command palette = Control + Shift + P or F1
  • Escape current line and move cursor down to new line. Creates new line without having to move cursor to end of current line = Control + Enter
  • Swap two lines = Alt + (Up or Down Arrow)
  • Duplicate line = Shift + Alt + (Up or Down Arrow)
  • Delete/"kill" line = Control + Shift + K
  • Preview markdown = Control + Shift + V
  • Toggle terminal = Control + `(backtick)
  • Split editor = Control + \
  • Switch between tabs = Control + Page Up/Down
  • Select current word = Control + D
  • Select current line = Control + L
  • Multiple cursors = Control + Alt + (Up or Down Arrow)
  • When editing HTML files in VSCode, we can just type the element name and the full element with tags < > will be generated
  • We can also generate multiple tags at once (ex: li*3 generates three "list item" elements)

  • Chrome Hotkeys:
  • Refresh page = Control + R
  • Open console = Control + Shift + J
  • Open elements = Control + Shift + C
  • Forward tab = Control + Tab
  • Backward tab = Control + Shift + Tab
